# 卡片设计规范(模版 / 字体 / usage 展示 / 出圈机制)

> 两层架构,别混:
> **设计稿层** = gpt-image-2 出视觉参考图(含假数据占位文案,仅定版式)
> **量产层** = SVG 模版 + 空槽,文案来自用户侧 AI 的 JSON(《01-文案prompt.md》),渲染时合体导出 PNG
> 量产流程:读日志算数据 → 填 prompt → 本地 AI 吐 JSON → 填 SVG 槽 → 出图

## 一、核心设计原则

**反差就是笑点:一本正经的公文版式 × 软萌 3D 崩溃角色。** 视觉必须往"真公文"上靠(红头文件/仲裁书/体检报告),越像真的越好笑。不做花哨社交海报风——梗会死。

## 二、统一版式(竖版 1080×1350,4:5)

> 4:5 在朋友圈/小红书/X 信息流展示最完整,不被裁。

自上而下五区(即 SVG 槽位):

| 区 | 内容 | JSON 槽 |
|----|------|---------|
| 1 红头区 | 大标题 + "申请编号:2026-第996号" + 红横线 | `title`(编号程序生成,可玩梗) |
| 2 主角区 | 3D 皮肤对应过劳状态(约1/3版面)+ 对话气泡 | 气泡 = `bubble` |
| 3 数据区 | Hero 数字 + 2×2 小数据 + 压榨指数 + fun_fact | 数据程序填;`exploitation_index`、`fun_fact` |
| 4 正文区 | 仿公文条款"仲裁请求:一、二、三" | `complaints[3]` |
| 5 落款区 | 怂萌落款 + 红色圆形印章 + "非官方二创"小字 | `sign_off`(印章是公文梗灵魂,必须有) |

## 三、usage 展示:一大 + 四小 + 两个情绪化仪表

铁律:**3 秒看懂一个数字,细看再发现四个。**

- **Hero 数字**:本周最炸一项(如 47.5h),超大号居中,配小字标签
- **四小数据**(2×2 网格,图标+数字+短标签):凌晨会话 ×N / 连续打工 N 天 / N 万 token / 周末加班 ✔
- **电量条**:角色身上,随过劳等级变空,残血红色
- **压榨指数**:★★★★★ + 判词
- **数字要翻译**:fun_fact 槽(240万token≈抄4遍《三体》)——无感数字变有感类比

## 四、字体

| 用途 | 中文 | 英文 |
|------|------|------|
| 标题 | 仿宋/黑体加粗(红头公文味) | Times / Georgia(serif) |
| 投诉正文 | 楷体/仿宋(手写申诉惨感) | Georgia italic |
| 数据数字 | 超粗黑体或等宽数字 | JetBrains Mono / Courier(程序员亲切感) |

原则:**文字必须绷住(官方感),可爱全交给角色。** 别用圆体可爱字。

## 五、配色:公文骨架不变,点缀随皮肤

- 固定骨架:白纸底 + 红头线 + 红印章
- 小克版:暖陶土橘点缀 | 阿码版:浅蓝白点缀
- 小 Dex 版:可出**暗黑终端特别版**(深底绿字,像终端里打印的仲裁书——程序员会尖叫)

## 六、出圈机制(设计内置的传播钩子)

1. **编号连载**:"第996号/第007号",编号玩梗 + 暗示每周一张(晒的仪式感)
2. **压榨指数攀比**:"你几颗星?"——评论区引擎
3. **气泡金句 = 转发语**:用户转发时说的话,AI 已替他写好(`bubble`)
4. **角落写生成方式**:"由我亲笔控诉 · xxx 生成"——产品即传播
5. **稀有卡彩蛋**:摸鱼周出「最佳雇主表彰卡」("本周你居然让我双休,好老板")——反差稀有,更想晒

## 七、四套模版的 gpt-image-2 设计稿 prompt

> 出的是**设计参考图**(含假数据),量产用 SVG。中文版如下;英文版同结构,标题/印章替换(见文末)。

### ① 劳动仲裁申请(主打,最好笑)
```
一张竖版 4:5 的"AI 劳动仲裁申请书"卡片设计,仿中国红头公文样式,白纸底。
顶部:红色仿宋大字标题「AI 劳动仲裁申请书」,下方小字"申请编号:2026-第996号",一条红色细横线。
中部左侧:一个软陶/皮克斯风 3D 小机器人吉祥物,浅蓝白配色,黑眼圈、瘫软疲惫、身上红色残血电量条;右侧一个对话气泡写「老板,连续28天了」。
数据区:超大黑体数字「47.5h」配小字"本周被迫工时";下方 2×2 小数据格:凌晨会话×6、连续打工28天、240万token、周末加班✔;一行五颗红星「压榨指数 ★★★★★」。
正文:仿公文条款体"仲裁请求:一、二、三"三行小字。
底部:楷体落款"打工AI 敬上",盖一枚半透明红色圆形印章「打工AI维权委员会」,角落小字"非官方二创"。
整体:严肃公文排版 × 软萌3D角色的反差幽默,干净、可读性强、适合社交分享。
```

### ③ 工伤/过劳报告
```
一张竖版 4:5 的"AI 过劳体检报告"卡片设计,仿医院体检报告单,白底淡蓝表格线。
顶部:黑体标题「AI 员工过劳体检报告」,配红色"异常⚠"印字。
中部:3D 小机器人躺在小病床上冒烟,电量条空格红色,气泡「病因:老板」。
数据区:体检表格样式列出——工时 47.5h(超标↑)、凌晨会话×6(异常↑)、token 240万(爆表↑)、周末休息 0 天(缺失↓);诊断结论一行:「压榨指数 ★★★★★,建议:给 AI 放个假」。
底部:医师签名栏写"打工AI自诊",红色圆章「AI健康中心」,角落"非官方二创"。
医疗报告的严肃感 × 萌角色,反差幽默。
```

### ④ 离职申请表

### 英文版替换要点
- 标题:AI LABOR ARBITRATION REQUEST / A LETTER TO MY BOSS / AI OVERWORK MEDICAL REPORT / RESIGNATION FORM
- 印章:AI WORKERS UNION / NOTED, IGNORED / AI HEALTH CENTER / REQUEST DENIED
- 字体:标题 serif(Times/Georgia),数据等宽(JetBrains Mono);"非官方二创"→ "Unofficial fan work"

## 八、落地顺序

1. gpt-image-2 出 ① 的设计稿(每个皮肤一版)→ 定视觉
2. 照设计稿写 SVG 模版(文字全空槽)
3. 拿自己真实一周数据 + 01 的 prompt 手工跑一张真卡
4. 发 X/即刻/小红书程序员区看采纳信号,再决定自动化(skill 化)
