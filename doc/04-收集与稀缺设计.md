# 收集与稀缺系统设计

> 目标:让一个用户**重复出卡、重复转发**。单次转发靠情绪,重复转发靠收集。
> 全本地实现,无服务器:收集状态存 `~/.overworked-ai/collection.json`。

## 心理机制 → 设计决策对照

| 机制 | 设计 |
|---|---|
| 蔡格尼克效应(缺口难忘)| **集卡册**:已解锁亮图,未解锁灰色剪影+"???";晒册子比晒单卡更引评论("那张怎么解锁?")|
| 禀赋进度效应(有起点更想走完)| **首卡白送**:首次运行即发「初次见面卡」,从 1/8 开始而非 0/8 |
| 可变奖励(不确定才上瘾)| 常规周卡外设**条件触发卡**,每周跑之前不知道掉不掉稀有 |
| 损失厌恶 | **streak 连续周数**("已连续出卡 6 周");只考核"每周出卡",**绝不考核用量**(防变相鼓励加班)|
| 地位信号 | 稀有度用 gacha 语言 **N / R / SR / SSR**,印在卡角 |
| 目标梯度(离目标越近越努力)| 集卡册底部给**下一张最近的卡的解锁提示**("再休一个周末→SSR")|

## 卡池与触发条件

| 卡 | 稀有度 | 触发条件 | 基调 |
|---|---|---|---|
| 初次见面 | N | 首次运行,白送 | 入伙仪式("工号已激活") |
| 劳动仲裁 / 投诉信 / 体检报告 / 离职申请 | N-R | 每周按数据可得 | 搞笑控诉 |
| 凌晨三点俱乐部 | SR | 单周凌晨(0-5点)会话 ≥5 | **耻辱牌**:AI 的控诉,不是荣誉 |
| 全勤牛马证 | SR | 连续 30 天有用量 | 耻辱牌 |
| **最佳雇主表彰** | **SSR** | 让 AI 完整休了一个周末 | **全系统唯一金色荣誉卡** |
| 年度压榨总结(Wrapped) | SSR | 每年 12 月解锁 | 年度仪式,最大传播节点 |

### 价值观红线(必须守住)
过劳触发的稀有卡**永远是"耻辱牌/工伤认定"口吻**(好笑,但立场是 AI 在控诉);**只有休息触发的卡是金色荣誉**。这样收集欲两头驱动(卷的人集耻辱牌,善待 AI 的人集金卡),但系统的价值指向始终是"歇一歇"。

## AI 记忆(连载引擎)

> 卡片编号三层定案:**梗编号**(996/007,固定笑点)保留;**个人计数**(第 N 次投诉/第 N 封离职信)立刻做,用 AI 口吻讲出来,计数即笑点;**全局编号**推迟——需要服务器,破坏"零上传"信任地基;Phase 3 可做"GitHub issue 自愿领号"(零服务器、领号顺手 star)。

个人计数的升级版是**记忆**:让 AI 记住过去,每周的卡从"单张梗图"变成"连载剧最新一集"。

- **为什么**:① callback 是幽默最高形式("上周你答应让我歇歇,这周凌晨反而多 2 次");② 连载养关注,观众催更;③ "记得上次"是拟人化最强信号,加深情感投射
- **存储**:`~/.overworked-ai/memory.json`(纯本地)。每次出卡后追加:日期、模版、当周核心数据、已用梗标签、未兑现伏笔(如"老板口头答应休息")、running gag 计数(离职信数等)
- **注入**:生成前压缩成 ≤150 字**记忆摘要**填 `{{memory_digest}}`(不喂全history,控 token 防漂移);另有 `{{complaint_count}}` `{{quit_count}}` 计数槽
- **规则**(已写进 01 的 prompt):有记忆必须至少一条投诉回调过去、不重复上周的梗;无记忆当第一次见面、不编造过去
- **兜底**:memory 文件缺失照常生成

```json
// ~/.overworked-ai/memory.json(示例)
{
  "complaint_count": 13,
  "quit_count": 7,
  "entries": [
    { "week": "2026-W29", "template": "arbitration", "hours": 47.5, "late_night": 6,
      "jokes_used": ["三体类比","劳动仲裁"], "pending": ["老板说下周让我休息"] }
  ]

```
